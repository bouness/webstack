🚀 WebStack Documentation

Your portable, isolated web development environment is running.

⚡ Quick Start

You can manage your stack via the GUI Manager (look for WebStack Manager in your app menu) or the CLI tools below.

# Start all services
webstack-start

# Open this documentation & test page
# Visit: http://localhost:8080

🐘 Switching PHP Versions

Multiple PHP versions are installed. Switch between them seamlessly:

# Switch to PHP 8.4
webstack-php 8.4

# Verify active version
webstack-php-cli -v

This updates the CLI, stops the old PHP-FPM pool, and starts the new one. Your web root (~/webstack-www) will immediately use the new version.

📁 File Structure

WebStack is split into two locations to keep binaries isolated from your data:

PathPurposeShared?
/opt/webstackCompiled binaries, dependencies, GUI managerYes (System-wide)
~/.webstackConfigs, databases, logs, PID filesNo (Per-user)
~/webstack-wwwYour web projects / document rootNo (Per-user)

⚙️ Configuration Files

  • Nginx: ~/.webstack/nginx/nginx.conf
  • PHP: ~/.webstack/php/<version>/php-fpm.conf & php.ini
  • MariaDB: ~/.webstack/mariadb/my.cnf
  • PostgreSQL: ~/.webstack/postgresql/data/pg_hba.conf

🔧 Troubleshooting

Port 8080 is already in use

Change the Nginx port in ~/.webstack/nginx/nginx.conf (change listen 8080; to your desired port), then restart Nginx.

GUI Manager won't open via SSH

The GUI requires a display server. Connect with X-forwarding: ssh -X user@host, then run webstack-manager.

Services won't start after reboot

WebStack does not auto-start on boot by design (it's a development environment). Run webstack-start after logging in.
